Openlab Services Api – 0.1.0

An api for the status of our services. All responses come in a data envelope which means that you get 2 root level objects. The 'data' section contains the payload of your request and 'meta' contains information about your request itself.

Services States

get / states.helloWorld Hello World

An endpoint to let you know everything is working correctly

{
  "meta": {
    "success": true,
    "messages": [],
    "status": 200,
    "name": "pinger",
    "version": "0.1.0"
  },
  "data": "ok"
}

get /all states.allStates All States

Gets the status of all services, each service comes back with information about it and whether it is currently live or not.

{
  "meta": {
    "success": true,
    "messages": [],
    "status": 200,
    "name": "pinger",
    "version": "0.1.0"
  },
  "data": {
    "openlab": {
      "name": "Main Site",
      "info": "OpenLab Wordpress",
      "link": "https://openlab.ncl.ac.uk",
      "state": {
        "online": true,
        "status": 200,
        "messages": []
      }
    },
    "digital_civics": {
      "name": "Digital Civics",
      "info": "Digital Civics Site",
      "link": "https://digitalcivics.io",
      "state": {
        "online": false,
        "status": 400,
        "messages": []
      }
    }
  }
}

get /service/:name states.serviceState Service State

Gets the state of a specific service. The response comes with information about the service and whether the service is running or not. If down, it may also return messages as to elude what is wrong.

nametypedescription
namestringThe service to request, available services can be found from /all
